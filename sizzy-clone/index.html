  • Everything is in perfect sync

    Scroll, click, or navigate to a different url, and all the devices will follow.

  • Scroll to any element

    Stay focused while working on a certain section of your app. Just write a selector for an element, and all devices will scroll to it even after you refresh the page.

  • Page Navigator

    Easily navigate every device by section or a heading. Just choose the direction and all the devices will navigate to it.

  • A developer's best friend

    If your local setup supports Hot Reloading you can instantly see all the changes you make across all of the devices. No more refreshing and manually switching to check if every device works.

  • Dev Tools

    Sizzy includes Chrome's dev tools so you can easily inspect and see what's going on with any device.

  • Universal Inspect Element

    Inspecting devices one by one can be a chore, so Sizzy includes Universal Inspect Element which allows you to inspect all devices at once. Nifty, huh?

  • Disable Network

    You can easily disable network per device, or for all the devices. This way you can test if your app works for users who are not online.

  • Touch Cursor

    Simulate a touch cursor and use all the gestures you would use on a real device. Touch, scroll, and swipe to your heart's content!

  • Organize devices

    Reorder, clone, delete, hide, and create devices to make Sizzy your own!

  • Add new devices

    If the built-in devices are not enough, you can always add new devices with custom dimensions, user agent, DPR, touch support, etc.

  • Manage Cache & Cookies

    Easily clear cache and cookies, or completely disable cache for all the devices

Frequently Asked Questions

You have questions, we have answers!

  • Will I get the same result as I would on a real device?

    Sizzy simulates the viewport and user agent of each device, so the results should be the same as what you would see on a real device.



    However, Sizzy cannot simulate different browser rendering engines, so there's a chance that there might be some minor differences when testing on a real device.

  • Why switch from OSS to a subscription model?

    The web app was running for 2 years and it had around 10.000 monthly users. Yet, it only got a few donations and a few open source contributions.



    We spent a lot of time on rewriting the app from scratch, adding tons of new features, and we have a lot of ideas for the future.



    In order to secure proper funding for development of the project, we had to switch to a paid model. We still chose a pricing model that is very affordable.

    Yes. Now when you click on the icon it will navigate Sizzy to the url you have opened in Chrome. You can download it here
